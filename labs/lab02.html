Lab 02 — Importing & Wrangling Data in R

The goals of this lab are to introduce you to a workflow you will use all term:

Import → Inspect → Wrangle Data (see also Fig. 1 of R4DS).

After completing this lab tutorial and assignment, you should be able to:

  • Import ecological data from common file types (.csv, .txt, .xlsx)
  • Inspect data quickly to catch problems early (names, missing values, column types)
  • Wrangle data using tidyverse verbs/functions
  • Explain the difference between long and wide data formats and reshape data between these formats
  • Write a clean, commented, reproducible R script that someone else can run

Equiped with these skills, you should be comfortable moving from I have a data file to I have a clean data frame I can analyze.

What is the tidyverse?

The tidyverse is a collection of R packages designed for data science. Its packages share an underlying design philosophy, a common grammar, and consistent data structures, so they work together smoothly. It includes packages you’ll use constantly in ecology, such as:

  • readr (import comma or tab delimited text files)
  • readxl (import Excel files)
  • dplyr (wrangle data with consistent “verbs”)
  • tidyr (reshape between long and wide formats)
  • ggplot2 (make data visualizations)

What is an R script?

An R script is a plain text file ending in .R that contains your code (and comments). Scripts are how we conduct reproducible data analysis tasks because they record every step of your analysis, can be re-run later to reproduce results, and make it easier to debug and collaborate. Learn more about R scripts in Chapter 6 of R4DS.

Best coding practices
  • Start with a header: name, date, purpose
  • Use clear object names (e.g., penguins_csv, not df1)
  • Use paths relative to the working directory (e.g., data/penguins.csv), never setwd()
  • Comment why you did something (not just what)
    • Comments can be added by using # before any text. R will ignore anything after #.
  • Use one tidyverse verb per line in pipelines for readability
  • Use spaces between either side of operators and after commas. For example, do this x <- 5 * y + mean(z, na.rm = TRUE) rather than this x<-5*y+mean(z,na.rm=TRUE)

Learn more about good coding practices in Chapter 4 of R4DS.
